1. Take your Bivec class from the last lab sheet and turn it into a template class, so that you could have a Bivec of other things apart from ints. Be careful to change only those occurrences of int that refer to the contents of the Bivec and not others, such as those that are acting as indices of the vector. Extend your main so that it uses a Bivec of string or double as well as a Bivec of int.

2. Retrieve your date program. Use cut-and-paste to put the class definition(s) into a file called date.h and the function definitions (but not main) into another file called date.cpp. In the date.cpp file, put #include "date.h" in the place where the class definition(s) used to be.

Now compile date.cpp using bcc32 with the -c -odate options, so as to produce a file of object code called date.obj: 

bcc32 -c -odate date.cpp
Check that date.obj now exists. (Note, use bcc32 for this exercise, not the usual bcpp. This is because bcpp renames all the object files to temp.obj, which we do not want for this exercise.)

Now take your Bivec program and insert #include "date.h" just after using namespace std; Extend main so as to include a Bivec of Date, e.g. 

Bivec<Date> dates(-1,5,Date());
// the default constructor is supplied as the third argument so as to provide
// an initial value for the dates in the Bivec
Include some code to test your Bivec of Date. Supply both bivec.cpp (or whatever you have called your program) and date.obj to the (bcc32) compiler so that the linker can link them together to generate the executable, e.g. 
bcc32 bivec.cpp date.obj
