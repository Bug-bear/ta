1.       The function scalar_product , as described in Exercise P9.1, takes two vectors (of the same size) of doubles as its parameters and returns the value a[0]*b[0] + a[1]*b[1] + ...
Write this function. Then create two vectors in a main function and pass them to your function to test it. The file N:\C++\doubles contains 20 floating-point numbers that you can use to set up your vectors. (If you put the first value into a[0], the second into b[0], the third into a[1] and so on, your output, to two decimal places, should be 2170.76.) Give some thought to how you pass the vectors.

 

The following three exercises all use this sequence of integers as example input:
1,   4, 9,   16, 9, 7,   4,   9,   11

It’s useful initially to use the same input as the examples so that you can check your output.    To avoid typing them in each time you run one of the programs, create a file in your directory containing just this line of integers, and then run your programs using this file. For each function, think carefully about the appropriate way to pass the vector to the function.

2.    The alternating_sum function, as described in Exercise P9.2, takes a vector of integers as its parameter, and returns the value a[0]-a[1]+a[2]-a[3]+a[4]-... Create a vector in main with the set of values given above, and then pass it to your function. Your main function should output the result (which should be -2).

3.    The reverse procedure, as described in Exercise P9.3, takes a vector of integers as its parameter and reverses the order of the elements; a vector containing 3, 12, 5, for example, would become 5, 12, 3. This procedure should actually change the contents of the vector, not just output them in reverse order. In fact, there should be no output in the procedure.  Build your vector in main, pass it to the procedure and then output the reversed vector in main.

4.    The function append, as described in Exercise P9.4, takes two vectors of integers and returns a vector consisting of the first one with the second one joined onto the end. Build two vectors in main (for example a vector of 0's and a vector of 1's), pass them to the function and then output the return value in main.

6.       Write a program that reads a line of text. It then prompts the user for a character to be deleted from the line and replaces all occurrences of that character with an asterisk and then outputs the line.
For example, if the input was "The quick brown fox jumped over the lazy dog"  and the character to be deleted was 'e', the output would be "Th* quick brown fox jump*d ov*r th* lazy dog"

7.       Write a program that reads five lines into a vector of strings and then displays the contents of the vector with each line preceded by its line number (1..5). Then prompt the user for a line number and remove that line from the vector, without changing the order of the remaining elements. Display the modified vector in the same format as above (except that this time the lines will be numbered 1..4). Use procedures to display the vector and remove the line.
(A procedure to remove elements from a vector is described in Horstmann p404 2nd ed., p341 3rd ed.)

Write a program to test the fairness of the random number generator, i.e. whether each possible value is generated approximately equally often.
Your program should ask the user:

·         how many random numbers should be generated (e.g. 1000)

·         what is the lowest number in the range (e.g. 1)

·         what is the highest number in the range (e.g. 10)

Make a vector with high – low +1 elements, one for each possible outcome. Call the random number generator the specified number of times. For each value v that it returns, increment the element of the vector with the subscript [v – low]. (You need to make this adjustment because the vector subscripts fall in the range 0.. high – low.)

After all of the numbers have been generated, output the results.

Below is sample output for 1000 numbers generated in the range 1..10.

  1 96
  2 101
  3  103
  4 116
  5 96
  6 109
  7 89
  8 87
  9 109
 10 94

To generate the random integers you should use the rand_int function (see Horstmann; p. 282 3rd ed. (p. 267 2nd ed.))

You will get the same results each time you run the program unless you also use the rand_seed function (same reference).
